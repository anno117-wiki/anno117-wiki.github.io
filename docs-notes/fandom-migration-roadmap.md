# Fandom日本語Wiki新設ロードマップ

## 方針

- **Fandom** → ゲーム情報Wiki（コミュニティが育てる）
- **GitHub Pages** → 計算機のみ維持（自分が管理）
- FandomからGitHub Pages計算機へURLリンクで連携

---

## Phase 1: 開設準備（1〜2日）

- [ ] Fandomアカウント作成
- [ ] `anno117-ja.fandom.com` でWiki新設申請
- [ ] 基本設定（言語・ロゴ・テーマカラー・説明文）
- [ ] カテゴリ・ナビゲーション構造の設計

## Phase 2: テンプレート整備（1〜2週間）

- [ ] Infoboxテンプレート作成（商品・建物・アイテム用）
- [ ] カテゴリツリー設計
- [ ] スタイルガイド策定（記事の書き方ルール）
- [ ] メインページデザイン

## Phase 3: コンテンツ移植（最大の山）

現在のVitePressコンテンツをMediaWiki形式に変換する。

| コンテンツ | 移植方法 | 優先度 |
|--|--|--|
| ガイド・概要記事 | Markdown → Wikiテキスト手動変換 | 高 |
| 商品一覧（6分類） | 個別記事×商品数 | 高 |
| 建物効果（173件） | Infobox記事化 | 中 |
| アイテム一覧（421件） | 個別記事化 | 中 |
| 住民層・地域情報 | 記事化 | 高 |
| 生産チェーン図（Mermaid） | 画像書き出し or 手描き図に変換 | 低 |

- [ ] 優先度「高」のコンテンツから順に移植
- [ ] 計算機へのリンクをFandomに設置（`/calculator/` URL）

## Phase 4: 計算機との分離整理

- [ ] GitHub Pagesの計算機にFandom WikiへのリンクURLを追加
- [ ] FandomのメインページとナビにGitHub Pages計算機URLを掲載
- [ ] VitePressのwiki部分は段階的にFandomへ誘導（リダイレクト等）

## Phase 5: コミュニティ運営体制

- [ ] 編集ガイドライン記事を作成
- [ ] 貢献者向けテンプレートの説明ページ作成
- [ ] SNS・ゲームコミュニティへの告知
- [ ] モデレーター体制の検討

---

## 廃棄・維持の整理

| 資産 | 判断 |
|--|--|
| Vueコンポーネント | 廃棄（Fandom管理外） |
| JSONデータ・公式データ | 維持（計算機で引き続き使用） |
| Mermaid図 | 画像として再利用 or 廃棄 |
| テーマ・CSS | 廃棄（FandomのスキンUI） |
| 計算機SPA | 維持（GitHub Pages） |
| VitePressのMarkdown記事 | 移植後に廃棄 or 開発用として残す |

---

## 注意点

- ゲームが新しいうちにコミュニティが形成されやすいため、早めの開設が有利
- アイテム421件・建物173件など件数が多いため、移植はコミュニティと分担する前提で進める
- 計算機との完全統合は失われる（URL誘導による連携のみ）
