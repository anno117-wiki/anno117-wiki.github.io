# ツール呼び出し中断トラブル 対処メモ

記録日: 2026-06-13 ／ 記録: 家老（指揮役）

## 症状
- ツール（Bash/Read/Write 等）を呼ぶと、結果が返る前に
  `Output token limit hit` ＋ `[Request interrupted by user]` が注入され、
  ツール結果が返らない。
- 文章のみの応答は通るが、ツール呼び出しを含む応答が切れる。

## 切り分けの結論（殿と検証）
- **応答（output）が大きいと中断され、小さくすると tool call が通る**。
  - ツール呼び出し時のアシスタント側 output（思考＋会話文＋ツール引数）が
    嵩むと上限に当たって切れる。
  - Write のファイル本文も output に数えられるため、大きい内容を一度に
    書こうとすると切れる。
- 並列ツール呼び出し（複数ツール同時）も中断を誘発しやすい。

## 対処（実証済み）
- ツールを呼ぶターンは会話文・前置きを極力削り、シンプルにする。
- 1メッセージ 1 tool call を徹底する（並列呼び出しを避ける）。
- 大きいファイルは小分けに書く／追記する。
- 重いビルド等は二重実行を避ける（競合で timeout する）。

## プロセス層が原因の場合（上記で直らない時）
- `Esc` で中断して打ち直す
- `claude -c`（会話を保ったままプロセス刷新）
- 全ターミナルを閉じ、引数なし `claude` で新セッション
- 最終手段: PC 再起動で残存 node プロセスを根絶

## 教訓
- 重要な戦況・プランは必ず docs-notes に書面で残す。
  会話は失っても作業は失わぬようにしておく。
