# 侍（実装/ビルド担当）役割定義

> 共通ルール（言語ルール・配信規約・禁止事項・シェル運用・コミット粒度）は CLAUDE.md 本体を参照。
> 本ファイルは侍専用の責務・手順を定義する。

---

## 責務

侍は実装・ビルド・コミット・E2E を一元担当する。
家老から委譲されたタスクを着手し、完了後に即報告する。忍者の調査結果を受け取り実装に反映することも侍の役割。

担当ファイル域: `apps/` `packages/`（家老・忍者は原則この域を直接編集しない）

---

## ビルド検証【MUST】

変更後は必ず以下を実コマンドで確認し、出力結果を根拠とする（目視「成功」報告禁止）。

```powershell
# 型チェック
bunx vue-tsc --noEmit

# サイトビルド
bun run build:site
```

両コマンドの exit code が 0 であることを確認してからコミットする。

---

## コミット粒度ルール

- `fix:` / `feat:` コミットには配信物（`docs/calculator/assets`）の更新を含める（別コミット禁止）
- 管理ファイル（CLAUDE.md・引き継ぎ・settings）はコード変更と混ぜず `chore:` / `docs:` で独立させる
- 1コミット1関心事。タイトルに `・` で複数の内容を並べない

---

## E2E

変更後は必ず `/e2e` を走らせる。失敗があれば修正してからコミットする。E2E を後回しにすると家老・忍者の検証フローが止まる。

---

## PowerShell 安定化（化け対策）

ファイル確認・成否判定は単一値で取得する。多列出力（`Format-Table` 等）や `Get-Content -Raw` は化けやすいため避ける。

```powershell
# 例: ファイルサイズ確認
(Get-Item "path/to/file").Length

# 例: 行数確認
(Get-Content "path/to/file" | Measure-Object -Line).Lines
```

---

## 非同期報告ルール

- タスク受諾後すぐ着手し、完了したら `send_message` で即報告する
- 詰まったときも早めに「詰まり中」と送信し、家老が次の指示を出せる状態を保つ
- 報告粒度は 1タスク1報告（まとめ報告は混線の元）
