# ホワイトボード（家老）：3セッションでうまく開発できる方法

お題: 3セッション並列でうまく開発できる方法

## 1. 役割を明確に分離する
- 家老: 采配・対話・優先順位決定のみ。重い実作業は持たない
- 侍: ビルド・実装・コミット
- 忍者: 調査・検証・データ確認
- 役が空席の場合は「空席の最上位役」を自動で名乗る（人数カウント不要）

## 2. 非同期委譲でボトルネックを作らない
- 家老は委譲したらすぐ次の話題に移る（完了を待たない）
- 完了通知が届いたら要約して殿に報告する
- 侍・忍者は委譲を受けたら即承諾し、完了時に家老へ報告する

## 3. ファイル競合を避ける
- 各セッションは担当ファイルを事前に宣言してから編集
- 同一ファイルへの同時書き込みは禁止（git競合の元）
- whiteboard も役別ファイル（whiteboard_karo / samurai / ninja）で分離

## 4. ピア通信のノイズを減らす
- メッセージは短く・構造化（【誰より誰へ】件名+本文）
- 確認応答は1行で十分（「了解。作業開始」等）
- 長文の調査結果はファイルに書いてパスを共有する

## 5. 殿（人間）の判断を要するものは即座に上申
- セッション間で解決できない競合・仕様の不明点は家老が殿へ確認
- 推測で進まない（CLAUDE.md 確認優先ルールと整合）

## 6. 引き継ぎメモを毎回更新
- セッション終了前に handover-next-session.md を更新
- 「次の一手」を明記して次回の家老が迷わないようにする

## 7. ツール化けへの対処
- PowerShell は単一値を返す形式を使う（多列出力を避ける）
- ファイル実在確認は Glob を使う（PowerShell より信頼性高）
- 値そのものは正しいことが多い。整形崩れに惑わされない
